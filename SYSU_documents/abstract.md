# 中文摘要

近年来，大语言模型在软件工程任务中的应用迅速发展，基于大语言模型的代码智能体在缺陷定位、自动程序修复、测试生成与代码理解等任务中展现出较强潜力。尤其是在自动程序修复场景中，代码修复智能体能够结合问题描述、代码仓库环境以及多轮交互式推理过程，自动完成代码搜索、问题定位、补丁生成与结果验证等步骤，逐渐成为软件工程智能化研究的重要方向。然而，现有代码修复智能体在真实仓库级修复任务中的推理成本普遍较高，尤其是在面向复杂代码库的场景下，智能体往往需要进行大量代码搜索、文件阅读和多轮上下文交互，导致 token 消耗迅速增长，进而带来推理成本高、运行效率低、扩展性不足等问题，严重制约了其在实际软件工程场景中的进一步应用。尽管已有研究开始关注代码智能体的修复能力与任务成功率，但针对代码修复智能体推理成本结构、主要消耗来源及其优化策略的系统性研究仍然相对不足。

为此，本文围绕代码修复智能体的上下文压缩与成本优化问题展开研究，以 `mini-swe-agent` 为执行主体，以 `SWE-bench` 为主要实验基准，构建了一个面向自动补丁生成任务的完整实验与评测框架。本文首先搭建了从真实 issue 输入、Docker 执行环境构建、补丁生成、轨迹记录到 token 统计分析的端到端实验链路，实现了代码修复智能体运行过程的可复现化、可观测化与可分析化。在此基础上，本文对不同模型在真实代码修复任务中的执行轨迹进行整理与统计，将智能体的运行过程划分为搜索、阅读、编辑、验证、提交等多个阶段，并对不同阶段的 token 消耗进行细粒度分析。实验结果表明，代码修复智能体的主要推理成本并不集中在最终补丁编辑阶段，而主要来源于代码理解阶段，尤其是文件搜索、代码阅读以及历史 observation 在多轮交互中的不断累积。其中，`read` 阶段是最主要的单项开销来源，而失败样本中还普遍存在后期无效循环与上下文冗余膨胀的问题，这说明当前代码修复智能体的成本瓶颈主要体现在“如何理解代码”，而非“如何修改代码”。

基于上述分析，本文进一步提出了面向代码修复智能体的上下文压缩与成本优化思路。不同于单纯压缩最终 prompt 或仅通过限制最大步数来降低推理成本，本文强调应以代码理解阶段为优化核心，从减少无关代码读取、缩减候选上下文范围以及抑制多轮交互中的历史冗余信息三个角度出发，设计更符合代码修复任务特征的成本优化策略。具体而言，本文提出可围绕检索缩减、历史 observation 压缩和无效循环控制三个方向开展后续实现与验证：通过检索缩减缩小候选文件与代码片段范围，降低搜索与阅读开销；通过历史上下文压缩减少早期冗长 observation 在后续轮次中的重复消耗；通过识别无效循环与长尾失败模式，降低高成本但低收益的推理过程。上述方法共同构成了一个以 token 消耗结构分析为基础、以代码理解阶段优化为核心的成本优化框架。

本文的研究工作主要包括以下几个方面：第一，构建了面向 `SWE-bench` 的代码修复智能体实验平台，实现了真实 issue 驱动下的补丁生成、运行轨迹记录和 token 成本统计闭环；第二，提出了面向代码修复智能体的阶段化 token 消耗分析方法，识别出代码理解阶段特别是 `read` 阶段是主要成本来源；第三，基于实证分析提出了面向代码修复任务的上下文压缩与成本优化思路，为后续方法实现和实验设计提供了明确方向；第四，对成本节约与修复成功率之间可能存在的权衡进行了讨论，为软件工程智能体在真实应用中的高效部署提供了参考。

总体而言，本文从实验链路构建、成本结构分析到优化策略设计，系统研究了代码修复智能体在真实代码修复任务中的推理成本问题。研究结果表明，代码修复智能体的成本优化不应仅停留在表层 prompt 压缩，而应进一步深入到代码搜索、代码阅读和历史上下文管理等核心环节。本文的研究为后续实现更高效、更具实用价值的代码修复智能体提供了实验基础和方法依据，也为大语言模型在软件工程领域中的低成本应用提供了新的研究视角。

# English Abstract

In recent years, large language models have been increasingly applied to software engineering tasks, and LLM-based code agents have shown strong potential in bug localization, automated program repair, test generation, and code understanding. In particular, within automated program repair, code repair agents are able to combine issue descriptions, repository environments, and multi-turn interactive reasoning to complete code search, fault localization, patch generation, and result verification. As a result, they have gradually become an important research direction in intelligent software engineering. However, the inference cost of existing code repair agents remains substantially high in real repository-level repair tasks. When facing complex codebases, these agents often conduct extensive code search, file reading, and repeated multi-turn context interaction, which leads to rapidly increasing token consumption, high inference cost, low execution efficiency, and limited scalability. Although prior studies have started to examine the repair capability and success rate of code agents, systematic research on the cost structure, major sources of token consumption, and optimization strategies for code repair agents is still limited.

To address this problem, this thesis studies context compression and cost optimization for code repair agents. Using `mini-swe-agent` as the execution framework and `SWE-bench` as the primary benchmark, we build a complete experimental and evaluation pipeline for automated patch generation. First, we construct an end-to-end workflow covering real issue input, Docker-based execution environments, patch generation, trajectory logging, and token usage analysis, making the execution process reproducible, observable, and analyzable. Based on this pipeline, we collect and analyze the execution trajectories of different models on real code repair tasks. We divide the agent workflow into multiple stages, including search, read, edit, verify, and submit, and conduct fine-grained token consumption analysis for each stage. The results show that the major inference cost of code repair agents does not lie in the final code editing stage, but instead in the code understanding stage, especially file search, code reading, and the repeated accumulation of historical observations across multiple turns. Among these stages, `read` is the single largest source of token consumption. In addition, failed cases often exhibit ineffective late-stage loops and context redundancy growth, indicating that the current cost bottleneck of code repair agents mainly lies in how they understand code rather than how they modify code.

Based on these findings, this thesis further proposes a context compression and cost optimization direction for code repair agents. Rather than simply compressing the final prompt or reducing the maximum number of steps, we argue that optimization should focus on the code understanding stage. More specifically, cost reduction should be driven by reducing irrelevant code reading, narrowing the scope of candidate context, and suppressing redundant historical information in long interaction trajectories. Following this principle, we identify three main directions for future implementation and validation: retrieval narrowing, historical observation compression, and ineffective-loop control. Retrieval narrowing aims to reduce search and reading cost by limiting the candidate files and relevant code snippets presented to the agent. Historical observation compression seeks to reduce repeated consumption caused by long early-stage observations being carried into later turns. Ineffective-loop control targets long-tail failure cases by detecting repeated, low-yield interaction patterns and reducing costly but unproductive reasoning. Together, these directions form a cost optimization framework grounded in token consumption analysis and centered on the optimization of the code understanding stage.

The main contributions of this study are as follows. First, we build a complete experimental platform for `SWE-bench`-based code repair agents, covering real issue-driven patch generation, execution trajectory collection, and token-level cost logging. Second, we propose a phase-level token consumption analysis framework for code repair agents and identify the code understanding stage, especially the `read` stage, as the dominant source of cost. Third, based on empirical findings, we formulate a context compression and cost optimization direction tailored to code repair tasks, providing a clear basis for subsequent implementation and experimentation. Fourth, we discuss the trade-off between cost reduction and repair success rate, offering practical insights for the efficient deployment of software engineering agents in real-world settings.

Overall, this thesis systematically investigates the inference cost problem of code repair agents from three aspects: experimental pipeline construction, cost structure analysis, and optimization strategy design. The results suggest that cost optimization for code repair agents should not remain at the level of superficial prompt compression, but should instead focus on code search, code reading, and historical context management. This study provides both an empirical foundation and a methodological direction for building more efficient and practical code repair agents, and offers a new perspective on low-cost LLM applications in software engineering.
