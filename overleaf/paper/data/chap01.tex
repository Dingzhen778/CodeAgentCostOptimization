\chapter{绪论}

\section{研究背景与意义（Claude Code 测试同步）}
近年来，Web3~\cite{web3_wikipedia}频繁进入大众视野，被视为互联网演进的下一代范式。其兴起主要得益于加密货币的广泛流行和区块链技术的快速发展。与传统Web2不同，Web3注重去中心化~\cite{wang2022exploring}、用户数据自主所有权以及无需信任第三方的网络交互环境。Web3的概念最早可追溯至万维网~\cite{berners1994world}创始人Tim Berners-Lee~\cite{bernerslee_wikipedia}提出的“语义网~\cite{semanticweb_wikipedia}”设想，即在无需中心化治理的前提下实现数据与知识自由共享。此后，以太坊联合创始人Gavin Wood~\cite{gavinwood_wikipedia}进一步提出了在无需可信第三方介入的条件下提供分布式互联网服务的愿景，从而让用户能掌控自己的数据，也使Web3被形象地描述为“读-写-拥有”（Read-Write-Own）型网络。在这一网络中，用户不仅能够自由访问和创建内容，更能真正拥有并自主管理自身的数据与数字资产。


如今，Web3依托区块链~\cite{zheng2018blockchain}与智能合约~\cite{wang2018overview}技术，实现了去中心化存储、用户数据自主所有权以及可信自动执行的网络交互环境。其中，区块链技术构建了去中心化的网络架构，实现数据的存储、交互与验证均由分布式网络节点共同维护。同时，用户通过掌控私钥对自身数据与资产拥有完全的控制权，实现了数据自主所有权；而智能合约技术则使开发者能够在去中心化网络中部署自动化应用程序，这些程序能够按照预定义的规则自动执行数据变更与价值转移操作。区块链与智能合约技术的结合，促进了大量Web3项目的出现，涵盖了去中心化金融（DeFi）、非同质化代币（NFT）、链游（GameFi）~\cite{proelss2023gamefi}以及去中心化社交平台等多个应用场景。截至2024年12月，已有超过21,358个Web3项目的智能合约部署于各大区块链平台~\cite{dappradar_dapp_amount}，累计活跃用户超过2,460万~\cite{dappradar_UAW}，市场总价值达到990亿美元以上~\cite{dappradar_dapp_marketcap}。


尽管现实中已有许多基于智能合约构建的Web3项目，并吸引了庞大的用户群体和创造了可观的市场价值，但Web3的定义仍未形成统一明确的共识，公众对Web3的认知也仍处于初步阶段。此外，作为Web3项目后端逻辑的智能合约由于其去中心化及不可更改特性，面临着诸多安全威胁。例如，在著名的“The DAO~\cite{dao_hack_gemini}”事件中，攻击者利用智能合约特有的可重入漏洞窃取了大量资金。然而，目前的研究中缺少对Web3项目中的智能合约安全问题的全面调查，且部分研究过度关注开发者并不优先考虑的问题。例如，一些研究追求极致的Gas~\cite{marchesi2020design}优化，但对实际开发者而言，这类细微的gas节省往往不足以抵消代码可读性降低所带来的负面影响。正如Aave的开发者“miguelmtzinf”所指出的~\cite{github_example}，Gas优化来带的节省很小，反而可能增加开发者理解和维护代码的难度，因此放弃了对Gas优化的追求。这一现象表明，深入了解开发人员对各类问题及漏洞的实际看法，对于设计和开发符合真实项目需求的安全工具具有重要意义。此外，现有漏洞检测工具普遍存在准确率不足和误报率较高的问题，难以有效满足实际安全需求。


为弥补上述研究空白与不足，本文将对当前流行的Web3项目开展调查与分析，梳理Web3生态系统的组成以及Web3项目在代码层面和市场层面的流行趋势，同时分析现有Web3项目所部署的智能合约可能存在的安全漏洞，并调研开发者在实际开发过程中对不同类型的问题或漏洞的态度及处理策略。在此基础上，本文设计并实现了一套符号执行与大语言模型协作的Web3智能合约漏洞检测工具，以协助开发者识别Web3项目中潜在的安全漏洞。


本研究系统梳理了Web3生态系统的结构，全面调研了主流Web3项目中智能合约的典型漏洞类型及其开发者在实际修复过程中的策略与权衡。在此基础上，设计并实现了一套面向Web3场景的智能合约漏洞检测工具。方法上，本文创新性地引入多角色大语言模型协作机制，模拟攻击者、安全专家、业务专家与仲裁者等多类角色，从多视角对初步检测结果进行语义审计与验证，从而有效提升检测的准确性与可信度。该方法为Web3合约安全检测提供了实用方案，也为智能合约安全研究拓展了新的技术路径。






\section{国内外研究现状}
\subsection{Web3相关研究}
Qin Wang等人通过对现有Web3项目进行全面的分析和系统性的架构梳理，并提出了一个包含客户端、计算和存储三个核心组件的Web3架构框架~\cite{wang2022exploring}；他们进一步识别出十二种不同的架构设计模式，并采用场景驱动的方式对这些模式进行了系统性评估。此外，论文还深入分析了Web3所面临的安全性、可扩展性以及互操作性等挑战，并讨论了Web3生态系统未来发展的潜力与机遇。

Dan Sheridan~\cite{sheridan2022web3}等人从Web3的技术背景、生态架构与市场环境三个维度对Web3进行了系统性梳理。他们首先分析了Web3的定义和核心特征，并指出Web3的出现不仅是技术层面的演进，更代表了一种以用户数据自主性为核心的互联网文化变革；其次，文章讨论了Web3生态系统的关键构成要素，包括区块链基础设施、智能合约、Web3程序库和数字身份管理（钱包）等重要组件，进一步探讨了Web3应用场景所面临的监管缺失、非法信息传播与隐私保护等挑战；最后，作者还分析了Web3开发人员所需的技能特征以及Web3对未来软件开发市场可能产生的深远影响和机遇。

Shuai Wang~\cite{wang2018overview}等人系统地梳理了智能合约的架构、应用场景及未来发展趋势，详细介绍了区块链智能合约的基本框架、运行机制、平台和编程语言，分析了其在金融服务、预测市场和物联网等领域的典型应用，并探讨了智能合约当前面临的安全性、隐私保护等技术挑战。此外，他们还展望了未来智能合约与并行区块链技术相结合的发展前景，以期为后续智能合约研究提供参考和指导。

Guangsheng Yu等人提出了一种名为WEBTTCOM~\cite{yu2024toward}的框架，以解决当前Web2向Web3过渡过程中存在的兼容性和易用性问题。该框架通过设计一个后端解释器，使传统的Web2应用无需彻底重构即可与区块链驱动的Web3应用无缝集成，并实现了去中心化的数据管理、灵活的访问控制和统一的用户身份管理。此外，作者通过在ServiceNow平台上实现基于Hyperledger Fabric的全栈服务管理系统，验证了WEBTTCOM框架在实际应用中的有效性，并通过性能测试与开发人员调查表明，该框架能够有效提升开发效率，降低Web2应用向Web3迁移的技术门槛。然而，目前框架的应用案例仍局限于许可链平台，未来需要进一步探索其在以太坊等更多公链环境中的适用性。


\subsection{智能合约安全相关研究}
Loi Luu等人提出了以太坊智能合约中的数种新的安全漏洞类型，包括交易顺序依赖、时间戳依赖、异常处理不当以及重入漏洞。作者设计了名为Oyente~\cite{luu2016making}的符号执行工具，该工具能够通过分析以太坊智能合约的字节码自动识别这些安全漏洞。作者使用Oyente检测了19,366个真实的以太坊智能合约，其中8,833个合约被标记为存在安全隐患，包含造成巨大经济损失的“The DAO”漏洞，体现了工具在实际场景中的有效性。同时，作者还探讨了漏洞产生的根本原因，提出了一些建议用于改善以太坊智能合约的执行语义。

Sukrit Kalra等人研发了一款名为ZEUS~\cite{kalra2018zeus}的智能合约安全分析工具，用于验证智能合约的正确性和公平性。ZEUS结合抽象解释和符号模型检测技术，通过受约束的霍恩子句~\cite{de2022analysis}对智能合约进行快速验证。ZEUS能够检测多种常见的智能合约漏洞，包括重入漏洞、未检查的发送、失败的发送、整数溢出/下溢、区块状态依赖以及交易顺序依赖等问题。此外，ZEUS通过将Solidity智能合约源代码转换为LLVM中间代码形式进行分析，支持跨区块链平台及多种高级编程语言的合约验证。实验表明，在分析超过22,493个以太坊智能合约后，发现其中约94.6\%的合约（涉及资产超过5亿美元）存在安全问题，ZEUS在安全分析速度和准确率方面显著优于现有工具Oyente，且漏报为零。

Yinxing Xue等人通过对11714个真实世界的智能合约进行大规模实证研究，总结了五种常见的防止重入攻击的路径保护技术，在此基础上研发了一个跨合约、跨函数静态分析的工具Clairvoyance~\cite{xue2020cross}，用于检测智能合约中的重入漏洞。该工具通过构建跨合约控制流图并结合静态污点分析，能够有效识别跨合约调用中的潜在重入路径，并利用轻量级的符号分析来排除不可行的攻击路径，从而显著降低误报率。实验结果表明，Clairvoyance在17770个真实世界的合约上检测出了101个此前未被发现的重入漏洞，精确度和召回率均优于现有的主流静态和动态工具。

Zeqin Liao等人提出了一种基于字节码级静态分析的跨合约漏洞检测方法SmartDagger~\cite{liao2022smartdagger}。SmartDagger通过神经机器翻译模型从智能合约的字节码中恢复合约属性信息，并通过选择性地构建跨合约控制流图和数据流图，高效地识别跨合约漏洞。实验表明，相比Oyente、Slither、Osiris和Mythril等现有工具，SmartDagger在检测跨合约漏洞方面表现出了更高的精确率和召回率。此外，SmartDagger还在现实中发现了以往工具未能检测到的真实跨合约漏洞。

Michael Rodler等人针对智能合约中的可重入漏洞问题，设计并实现了一种名为Sereum~\cite{rodler2018sereum}的安全工具。Sereum无需修改现有智能合约代码，通过动态污点追踪技术在以太坊虚拟机上实时监测智能合约的执行流，并动态识别合约运行期间因状态不一致而导致的重入攻击行为。该工具能够有效检测并阻止传统的同函数重入、跨函数重入、委托调用重入、以及基于合约创建的重入等多种高级可重入攻击，并通过对以太坊主网上近7800万笔交易的评估，展示出较低的误报率（0.06\%）和较小的运行时开销（9.6\%）。然而，Sereum作为运行时监测工具，可能存在由于以太坊字节码缺乏字段敏感性、存储释放操作及构造函数回调等情况引起的误报问题。

杨慧文等人提出了一种基于软件度量的Solidity智能合约缺陷预测方法~\cite{yang2022solidity}。作者首先设计了SC-Sol度量元集，覆盖了Solidity语言特有的变量类型、函数结构与典型漏洞相关特征，并与传统面向对象度量元集COOP结合，形成COOP-SC-Sol联合度量体系。随后，作者从Solidity合约中提取度量信息，结合BSCSCS平台的检测结果构建缺陷数据集，并在此基础上应用多种回归与分类模型对合约缺陷数量与缺陷倾向性进行预测。实验结果表明，所提出的度量体系能够显著提升预测模型的性能，其中在F1-score指标上相较传统COOP模型提升了最高达8\%。此外，作者还研究了类不平衡问题对模型性能的影响，发现随机欠采样技术对改进预测效果具有显著作用。该工作为智能合约的质量保障提供了一种轻量化、可解释的预测路径，丰富了智能合约安全保障的研究视角。

黄静等人提出了一种基于代码控制流图的庞氏骗局智能合约检测模型~\cite{huang2025ponzi}。该方法首先将Solidity合约源码构建为控制流图，并借助Word2Vec提取其中的数据流与结构信息以构成节点特征表示。随后，作者引入图卷积网络与Transformer结合的特征学习模块，利用多头注意力机制捕捉控制流图中节点之间的复杂依赖关系。最后，通过多层感知机对提取的图特征进行分类，实现庞氏骗局合约的自动识别。实验在公开庞氏骗局合约数据集上进行了评估，结果显示该模型在准确率、精确率、召回率和F1分数等指标上均优于传统图学习模型及多种机器学习基线方法，F1分数达到0.87，显著提升了庞氏骗局合约的检测效果。该研究证明了将控制流语义与图神经结构融合可显著增强智能合约欺诈检测任务的性能。

崔展齐~\cite{cui2024survey}等人对智能合约安全漏洞检测领域的研究进展进行了系统性综述。作者构建了一个涵盖漏洞发现与识别、漏洞检测方法、数据集与评价指标的研究框架，并从五种主流技术路径出发对现有方法进行分类和分析，包括符号执行、模糊测试、机器学习、形式化验证与静态分析。特别是在漏洞类型分析方面，作者总结了41种典型漏洞模式，并将其归类为13大类，如重入漏洞、未检查外部调用返回值、整数溢出、区块信息依赖等。文中不仅评估了多种检测方法的优劣，还梳理了主流开源数据集与常用评估指标，为未来研究提供了系统性参考。该综述全面反映了当前智能合约安全检测技术的发展现状，具有较强的参考与指导价值。


\section{主要贡献与创新}
为了全面系统地梳理Web3生态系统的组成，深入分析当前作为Web3后端逻辑的智能合约面临的安全问题，并设计Web3智能合约漏洞检测工具有效识别漏洞，本文分阶段开展了以下研究工作：

首先，本文收集了当前流行的Web3项目，采用开放式卡片分类法将其划分为Web3基础设施与Web3应用程序两大类，并进一步细分出各自包含的若干子类以及介绍各子类的有代表性项目，明确了Web3生态系统的整体结构。此外，通过对GitHub代码库和区块链浏览器市场数据的分析，揭示了Web3项目在代码开发上的活跃程度与实际市场表现，确定了Web3项目在技术开发与市场应用中的流行趋势。

之后，为了明确Web3智能合约当前面临的主要安全威胁，本文通过爬取GitHub上Web3开源项目的安全问题报告，借助大语言模型进行自动化分析并加以人工审查，系统地归纳出了Web3智能合约领域内最常见的八种安全漏洞，即：访问控制漏洞、不安全随机数漏洞、整数溢出漏洞、重入攻击漏洞、区块信息依赖漏洞、拒绝服务攻击漏洞、未检查的外部调用漏洞和短地址攻击漏洞。同时，本文对开发者在实际漏洞修复与优化过程中的处理态度和方式进行了调研和分析，揭示出开发者在面对漏洞时的实际决策与权衡，并说明并非所有潜在的安全风险都被开发者视为优先关注的目标。

随后，本文针对上述八种漏洞分别进行了深入分析，具体从漏洞产生原因、防范方法和典型实例三个方面展开论述，并为每种漏洞设计了针对性的自动化检测方案。

最后，本文进一步设计并实现了一种基于符号执行与多角色大模型~\cite{naveed2023comprehensive}协作的Web3智能合约漏洞检测工具。该工具主要由预处理模块、控制流图~\cite{fritz1995overview}构建模块、符号执行模块、漏洞检测模块和多角色大模型协作验证模块组成。预处理模块负责智能合约编译后的解析与操作码序列提取；CFG构建模块对智能合约字节码进行基本块划分并构建控制流图；符号执行模块模拟合约的实际执行过程；漏洞检测模块结合预定义的漏洞特征自动进行漏洞识别。此外，本文创新性地引入多角色大模型协作验证模块，该模块通过大模型模拟了攻击者、安全专家、业务逻辑专家和仲裁者四个角色：攻击者负责审查是否存在攻击路径；安全专家负责审查安全措施和实践是否被采用；业务逻辑专家负责分析代码的业务逻辑；仲裁角色综合前三者的信息判断漏洞存在误报的可能，从而降低了漏洞检测过程中的误报率。

为了验证工具的有效性，本文通过对包含143个现实世界智能合约的基准数据集SmartBugs-Curated~\cite{smartbugs_curated}进行实验，评估了本文工具的有效性，实验结果表明该工具能够有效检测Web3智能合约中的常见安全漏洞，检测准确率达到93.85\%，召回率为89.71\%，F1分数为91.73\%。

本文的主要贡献与创新点可归纳如下：

（1）系统性地梳理了Web3生态系统的整体结构，并对其中各领域的代表性项目进行了深入介绍；

（2）揭示了Web3开发者在漏洞修复与优化上的决策倾向与实际权衡，指出并非所有安全风险均为开发者优先关注的目标；

（3）提出并实现了一种基于符号执行和多角色大模型协作的Web3智能合约漏洞检测方法，该工具在准确率（93.85\%）、召回率（89.71\%）和F1分数（91.73\%）上均优于现有主流工具如Mythril、Oyente和Slither。
