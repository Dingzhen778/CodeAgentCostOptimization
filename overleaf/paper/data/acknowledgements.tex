\section*{演示：Vibe Writing 工作流}

本页作为 Vibe Writing 能力的展示页，记录一次完整的"写作 + 画图 + 同步"对话流程，供同学参考。

\subsection*{1. 对话式驱动的协作模式}

在 Claude Code 环境下，写论文的工作模式从"打字 + 手画图"转变为"自然语言指令 + AI 执行"。整个流程由三端联动完成：本地文件系统、Overleaf 云端项目、GitHub 仓库，三者通过约定好的脚本（\texttt{ot-upload.py}、\texttt{ols}、\texttt{git}）保持一致。

典型的一次交互形如：

\begin{quote}
\textbf{用户}：帮我在 chap05 末尾加一张方法依赖图。\\
\textbf{AI}：好的，我用 \texttt{drawio\_generator.py} 画一张五层结构图，渲染成 PNG 和 PDF，放到 \texttt{figures/}，再在 chap05 里插入 \texttt{\textbackslash includegraphics} 引用。\\
\textbf{用户}：颜色太深了，换浅色系，方法层七个框用中文名。\\
\textbf{AI}：\emph{（修改脚本中的 Style 常量、替换中英文标签、重新渲染）}\\
\textbf{用户}：右侧有大片空白，看起来图只占了一角。\\
\textbf{AI}：\emph{（自动检测非白区域边界，裁掉多余画布，重新转 PDF）}
\end{quote}

整个过程，用户只需要用自然语言描述"我希望的结果是什么"，无需关心具体的 XML 语法、puppeteer 渲染参数、PIL 图像处理等底层细节。

\subsection*{2. 图片迭代示例}

以本论文方法依赖图的生成为例，图片经过了三轮迭代：

\begin{itemize}
    \item \textbf{v1}：五层结构初稿，但字号过小、箭头交叉严重。
    \item \textbf{v2}：调整层间距、增大字号，但"方法层"标签与最右侧方框重叠。
    \item \textbf{v3}：左移层标签位置、替换为浅色系、全部中文化，最终落地为论文插图。
\end{itemize}

每一轮只需要一句话描述要改什么，AI 负责修改 Python 源码、重新渲染、检查边界、覆盖到 \texttt{figures/}。版本化的好处在于，任何一版都能通过重新运行脚本完整复现，不再依赖"手动拖动节点"的记忆。

\subsection*{3. 同步自动化}

写完章节、改完图后，同步到云端只需要两条命令，整个过程可以通过一句"把 chap05 传到 Overleaf"触发：

\begin{enumerate}
    \item OT 协议单文件覆盖：\texttt{python3 ot-upload.py chap05.tex <doc\_id>}
    \item Git 版本化：\texttt{git commit -m "..." \&\& git push}
\end{enumerate}

二进制图片（PDF）由于 Overleaf REST 接口限制，需要手动在 Web UI 拖拽上传。这一约束也写入了 skill 文档，AI 在遇到图片同步时会主动提醒。

\subsection*{4. 可复现性}

上述所有流程均记录在项目根目录的 \texttt{how\_to\_vibe\_writing.md} 中，包含：凭证获取、目录约定、绘图脚本用法、PDF 转换坑点（如裁白边、伪 PDF 格式识别）、常见故障排查表。后续使用者只需按文档流程配置一次环境，即可复现同样的协作模式。
