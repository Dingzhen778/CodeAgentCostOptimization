\begin{abstract}\label{abstract}

  近年来，基于大语言模型的代码智能体在自动程序修复与仓库级问题求解中展现出显著潜力。
  然而，随着任务从函数级代码生成转向真实仓库级修复，智能体在多轮交互中反复搜索代码、
  读取大量上下文并累积历史执行轨迹，导致词元（词元）消耗随步数超线性增长，严重制约其工程部署可用性。
  
  本文以 \texttt{mini-swe-agent} 为执行框架、\texttt{MiniMaxAI/MiniMax-M2.5} 为推理模型、
  SWE-bench Verified 500 条实例为评测基准，构建了端到端实验平台，
  通过本地透明代理网关实现每步 词元 精确计量，保证成本数据的可复现性。
  
  在成本结构分析方面，本文将执行轨迹划分为搜索、阅读、编辑、验证、提交五个阶段进行细粒度归因。
  结果表明，代码阅读阶段约占总 词元 消耗的 50\%；代码修改阶段仅占 2--3\%。
  失败轨迹几乎总是耗尽最大步数预算（40 步），平均消耗约为成功轨迹的 2--3 倍，
  且占全部成本的约 83\%。成本优化的核心在于减少定位修复位置过程中的冗余搜索与阅读，
  而非压缩补丁生成阶段的输出。
  
  在方法设计方面，本文沿上下文压缩与工作流压缩两条正交路径，设计并实现了六种方法：
  文件级 RAG 检索（\texttt{rag\_topk}）、函数级裁剪检索（\texttt{rag\_function}）、
  LLMLingua 语义重排序（\texttt{llmlingua\_original}）、结构化工作流脚手架注入（\texttt{skill\_abstraction}）、
  静态跨任务经验记忆注入（\texttt{skill\_memory\_md}）及三阶段组合压缩（\texttt{hybrid\_llmlingua}），
  并与无增强基线（\texttt{baseline\_raw}）在统一框架下对比，评估维度涵盖解决率、平均 词元 消耗与单位成功成本（CPR）。
  
  实验结果表明，\texttt{skill\_memory\_md} 和 \texttt{rag\_topk} 综合表现最优，
  CPR 较基线分别降低 10.5\% 和 10.0\%，解决率分别提升 3.2 和 3.8 个百分点。
  通用文本压缩（\texttt{llmlingua\_original}）虽实现最大 词元 削减（$-$5.74\%），
  但因代码结构性信息损失导致解决率下降，CPR 改善仅 2.9\%，说明通用压缩难以直接适配代码修复场景。
  函数级裁剪（\texttt{rag\_function}）因截断跨文件上下文导致解决率低于基线，
  验证了仓库级修复需要保留结构性语义。
  
  本文的主要贡献包括：构建了基于 SWE-bench Verified 的可复现实验平台；
  提出了阶段化成本分析框架；实现并系统评估了六种优化方法；
  建立了以解决率、平均 词元 消耗与 CPR 为核心的成本—能力联合评估框架。
  研究表明，降低代码修复智能体推理成本的有效路径在于改善搜索效率与决策质量，
  而非简单压缩输入 词元，为后续高效代码智能体研究提供了实验基础与方法参考。

  \begin{abstract*}
  
  In recent years, large language models (LLMs) have demonstrated significant potential
  in software engineering tasks, particularly in automated program repair, code
  understanding, and repository-level issue resolution. However, as task complexity
  shifts from function-level code generation to real-world repository-level repair,
  the inference cost of code repair agents has emerged as a critical bottleneck. Agents
  must iteratively search code files, read large contexts, and accumulate execution
  histories across multiple rounds of interaction, causing 词元 consumption to grow
  super-linearly with the number of steps. This severely limits the practical
  deployability of such systems. Despite growing interest in agent capability and
  task success rates, systematic study of the cost structure and optimization strategies
  for code repair agents remains insufficient.
  
  This paper addresses the high inference cost problem of code repair agents through
  a systematic experimental and analytical study. Using \texttt{mini-swe-agent} as the
  execution framework, \texttt{MiniMaxAI/MiniMax-M2.5} as the reasoning model, and
  SWE-bench Verified (500 instances) as the evaluation benchmark, we construct a
  complete end-to-end experimental pipeline covering issue input, Docker-isolated
  execution, patch generation, trajectory recording, and per-step 词元 metering
  via a local transparent proxy gateway.
  
  For cost structure analysis, we decompose agent trajectories into five phases:
  search, read, edit, verify, and submit, and perform fine-grained 词元 attribution
  across phases. Results show that the \textit{understand} (code reading) phase
  accounts for approximately 50\% of total 词元 consumption, while the
  \textit{implement} (code editing) phase contributes only 2--3\%. Failed trajectories
  almost universally exhaust the maximum step budget (40 steps), consuming 2--3$\times$
  more 词元s than successful ones, and account for approximately 83\% of total
  inference cost. These findings reveal that the primary cost bottleneck lies in
  ``locating the correct repair site'' rather than ``generating the patch itself.''
  
  Based on these findings, we design and implement six optimization methods along two
  orthogonal paths: \textbf{context compression} and \textbf{workflow compression}.
  Context compression methods reduce the amount of code context injected before
  agent execution: file-level RAG retrieval with BM25 scoring (\texttt{rag\_topk}),
  function-level code trimming on top of RAG (\texttt{rag\_function}), and
  LLMLingua-based semantic re-ranking (\texttt{llmlingua\_original}).
  Workflow compression methods improve agent decision-making by injecting structured
  guidance: a six-step workflow scaffold (\texttt{skill\_abstraction}) and a static
  transferable skill memory derived from resolved instances (\texttt{skill\_memory\_md}).
  A three-stage combined method (\texttt{hybrid\_llmlingua}) integrates both paths.
  All methods are evaluated against an unaugmented baseline (\texttt{baseline\_raw})
  under a unified execution framework.
  
  Experimental results on 500 SWE-bench Verified instances demonstrate that
  \texttt{skill\_memory\_md} and \texttt{rag\_topk} achieve the best overall
  performance, reducing CPR (Cost Per Resolved instance) by 10.5\% and 10.0\%
  respectively while simultaneously improving resolve rates by 3.2 and 3.8 percentage
  points. The general-purpose text compression method (\texttt{llmlingua\_original})
  achieves the largest 词元 reduction ($-$5.74\%) but suffers from decreased resolve
  rate due to structural information loss in code, yielding only a 2.9\% CPR
  improvement. This confirms that general NLP compression techniques cannot be
  directly adapted to code repair without careful consideration of code structure.
  Function-level trimming (\texttt{rag\_function}) degrades performance below
  baseline by severing cross-function and cross-file dependencies, further validating
  that repository-level repair requires preserving structural semantics beyond individual
  functions. Analysis of exclusive-success instances reveals that different methods
  optimize distinct bottlenecks---file retrieval improves localization efficiency,
  skill memory improves decision quality, and workflow scaffolding accelerates
  convergence---demonstrating strong complementarity among methods.
  
  The main contributions of this paper are fourfold: (1) a reproducible experimental
  platform for code repair agents on SWE-bench Verified, with precise per-step 词元
  metering; (2) a phase-level cost attribution framework identifying code reading and
  failed trajectories as the dominant cost sources; (3) six optimization methods across
  context and workflow compression paths, systematically evaluated on 500 instances;
  and (4) a joint cost--capability evaluation framework based on resolve rate, average
  词元 consumption, and CPR, providing quantitative guidance for agent deployment
  and future research.
  
  Overall, this work demonstrates that effective cost reduction for code repair agents
  requires improving search efficiency and decision quality rather than naively
  compressing input 词元s. Controlling the cost of failed trajectories and further
  improving resolve rates represent the most impactful directions for future work.
  This research provides an experimental foundation and methodological reference for
  building more efficient code repair agents and advancing LLM-driven software
  engineering automation.
