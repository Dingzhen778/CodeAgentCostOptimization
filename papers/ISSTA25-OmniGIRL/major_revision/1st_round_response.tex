\begin{revmeta}[Thanks for the comments! In the revision, we have added an LLM agent-based approach, AutoCodeRover, as an additional  baseline. We choose this method because of its robust performance on the SWE-bench leaderboard \text{[Ref4]}. This approach uses a context retrieval agent to collect relevant information from the codebase and a patch generation agent to generate patches resolving the corresponding issue. Because this method is currently designed for Python, we have adapted it to multiple languages without changing its original design. This adaptation is referred to as AutoCodeRover-X. We evaluate this approach on OmniGIRL and find that the original conclusions do not change. First, we find that the AutoCodeRover method shows limited performance on OmniGIRL, with the highest resolve rate of 8.1\%. Second, we find that this method shows limited performance on issues with visual
input, with the highest performance of 5.3\%. Besides, we find that visual information can enhance the performance of AutoCodeRover-X on issues requiring image understanding. Third, we find that the AutoCodeRover method performs significantly worse on cross-file issues compared to single-file issues. Besides, we find that LLMs with the AutoCodeRover-X still show a high tendency to modify a single file on cross-file issues. Please refer to Comment 2.1 for the detailed response.]    
\end{revmeta}
